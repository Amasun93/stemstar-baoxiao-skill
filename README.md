# StemStar Baoxiao Skill

StemStar 报销材料整理与归档 skill，用于让 Codex 按统一规则整理发票、订单截图、支付截图、行程单等报销材料。

## 怎么使用

这个 skill 能用来：帮你检查报销材料是否齐全、核对发票/订单/支付截图金额是否一致、提醒缺票或金额差异、统一文件命名、按月份放入 `YYMM 待报销`，并在材料提报后归档到 `00 归档/YYMM 报销-总金额`。

请把这个链接复制给你的 AI：

```text
https://github.com/Amasun93/stemstar-baoxiao-skill
```

可以直接这样对 AI 说：

```text
请使用这个 skill 帮我整理报销材料：
https://github.com/Amasun93/stemstar-baoxiao-skill

我的需求是：检查材料是否齐全，核对发票/订单/支付截图金额是否一致，提醒缺票或金额差异，按金额和事项重命名，放入当月 YYMM 待报销；等我说已经提报后，再归档到 00 归档/YYMM 报销-总金额。
```

如果你的 AI 支持本地安装 skill，可以把这个仓库放到 Codex skills 目录下：

```powershell
git clone https://github.com/Amasun93/stemstar-baoxiao-skill.git "$env:USERPROFILE\.codex\skills\stemstar-baoxiao-skill"
```

## 适用场景

- 检查报销材料文件名是否规范
- 核对发票、订单截图、支付截图、行程单金额是否一致
- 提醒缺少发票、行程单或存在金额差异的材料
- 把新材料整理到当月 `YYMM 待报销`
- 按事项拆分为 `金额 事项名称`
- 核对发票金额、实付金额、订单金额是否一致
- 已提报后移动到 `00 归档/YYMM 报销-总金额`
- 清理旧归档，让月份和金额更容易查找

## 推荐目录结构

报销根目录建议保持简单：

```text
D01 报销/
├─ 00 归档/
├─ 2607 待报销/
└─ 报销细则.pptx
```

不要在根目录再使用 `待整理`、`待处理`、`6月` 这类泛化文件夹。未提报材料统一放入 `YYMM 待报销`。

## 命名规则

待报销或已归档的月份文件夹：

```text
YYMM 待报销
YYMM 报销-总金额
```

事项文件夹：

```text
金额 事项名称
```

细项文件夹：

```text
金额 事项类别
```

文件名：

```text
金额 事项_对象或路线_材料类型.ext
```

示例：

```text
2607 待报销/
└─ 1192.49 武汉讲座/
   ├─ 1010 武汉讲座机票/
   │  ├─ 550 武汉讲座机票_上海-武汉_发票.pdf
   │  ├─ 550 武汉讲座机票_上海-武汉_支付截图.jpg
   │  ├─ 460 武汉讲座机票_武汉-上海_发票(票面501).pdf
   │  └─ 460 武汉讲座机票_武汉-上海_支付截图.jpg
   └─ 182.49 武汉讲座打车/
      ├─ 182.49 武汉讲座打车_订单截图.jpg
      └─ 92.29 武汉讲座打车_武汉大学-天河机场_行程单(票面77.29).pdf
```

安装后，在 Codex 里可以这样说：

```text
用 stemstar-baoxiao-skill 帮我整理这些报销材料
```

或者：

```text
这些材料已经提报了，帮我归档到当月报销文件夹
```

## 注意

- 发票票面金额和实付金额不一致时，优先按用户确认的报销口径处理，并在文件名里保留票面金额说明。
- 支付或订单金额大于发票合计时，必须提醒具体缺口，例如 `返程支付截图 415，现有发票 284+87，缺 44 元发票`。
- 桌面或临时目录里的原始材料通常只复制进报销目录，不直接删除。
- 无法确认月份、金额或对应关系的材料，放入当月 `YYMM 待报销/待确认材料`。
